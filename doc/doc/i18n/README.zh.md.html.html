<div class="pilwrap" id="sparkles">
  <h2>
    <a href="#sparkles" name="sparkles" class="pilcrow"></a>
✨ 功能
  </h2>
</div>
<p>当其他平台选择闭源路线时，我们决定采用开放源代码。当其他平台选择云服务时，我们决定提供本地构建器提供更好的开发体验。我们喜欢在Budibase上做不同的事情。</p>
<ul>
<li><strong>构建和发布真实的软件</strong> 与其他平台不同，使用Budibase可以构建和交付单页应用程序。Budibase应用程序具有增强的性能，并且可以进行响应式设计，从而为您的用户提供出色的体验。</li>
<li><strong>开源且可扩展</strong> Budibase是开源的。Builder使用AGPLv3许可，Server使用GPLv3，Client使用MPL。这会让您充满信心，Budibase将永远存在。您还可以针对Budibase进行修改或对其进行分叉，并根据需要进行更改，以提供开发人员友好的体验。</li>
<li><strong>连接数据库或直接开始</strong> Budibase从多个来源（包括MongoDB，CouchDB，PostgreSQL，mySQL，Airtable，Google Sheets，S3，DyanmoDB或REST API）提取数据。与其他平台不同，使用Budibase可以从头开始，创建没有数据源的业务应用程序。<a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">新的数据源需求</a>。</li>
<li><strong>使用强大的预制组件设计和构建应用程序</strong> Budibase开箱即用，具有精美设计，功能强大的组件，您可以使用它们像构建基块来构建UI。我们还提供了许多您最喜欢的CSS样式选项，因此您可以发挥更多的创意。<a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">新的组件需求</a>。</li>
<li><strong>自动化流程，与其他工具集成，并连接到Webhooks</strong> 通过自动化手动流程和工作流来节省时间。从连接到Webhook，到自动发送电子邮件，只需告诉Budibase要做些什么，然后让它为您工作。您可以<a href="https://github.com/Budibase/automations">在此处</a>轻松地<a href="https://github.com/Budibase/automations">为Budibase创建新的自动化，</a>或<a href="https://github.com/Budibase/automations">在此处</a><a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">新的集成需求</a>。</li>
<li><strong>云托管和自我托管</strong> 用户可以自行托管（请参见下文），或使用Budibase托管其应用。目前，我们的云托管产品仅限于免费套餐，但我们会在将来进行更改。对于大量使用，我们建议用户进行自我托管。</li>
</ul>

    <a href="#budibase-1" name="budibase-1" class="pilcrow"></a>
🏁 Budibase入门
  </h2>
</div>
<p>Budibase构建器在Mac，PC和Linux上的Electron中运行。请按照以下步骤开始：</p>
<ul>
<li>[ ] <a href="https://portal.budi.live/signup">注册Budibase</a></li>
<li>[ ] 创建用户名和密码</li>
<li>[ ] 复制您的API密钥</li>
<li>[ ] 下载Budibase</li>
<li>[ ] 打开Budibase并输入您的API密钥</li>
</ul>

<div class="pilwrap" id="-2">
  <h2>
    <a href="#-2" name="-2" class="pilcrow"></a>
🤖 自托管
  </h2>
</div>
<p>Budibase希望确保任何人都可以使用我们开发的工具，并且我们知道很多人需要能够在自己的系统上托管他们制作的应用程序——这就是为什么我们决定尝试使自托管服务变得如此简单的原因！</p>
<p>当前，您可以使用Docker或Digital Ocean托管您的应用程序。可以在<a href="https://docs.budibase.com/self-hosting/introduction-to-self-hosting">此处</a>找到有关自我托管的文档。</p>

<div class="pilwrap" id="-5">
  <h3>
    <a href="#-5" name="-5" class="pilcrow"></a>
存储库的组织方式
  </h3>
</div>
<p>Budibase是由lerna管理的项目。Lerna管理budibase软件包的构建和发布。在较高的层次上，这里是构成Budibase的软件包。</p>
<ul>
<li><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/builder">packages/builder</a>-包含budibase构建器客户端苗条应用程序的代码。</li>
<li><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/client">packages/client</a>-在浏览器中运行的模块，负责读取JSON定义并从中创建生动的Web应用程序。</li>
<li><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/server">packages/server</a>-budibase服务器。该Koa应用程序负责为构建器和budibase应用程序提供JS服务，并提供与数据库和文件系统交互的API。</li>
</ul>
<p>有关更多信息，请参见<a href="https://github.com/Budibase/budibase/blob/HEAD/.github/CONTRIBUTING.md">CONTRIBUTING.md</a></p>
