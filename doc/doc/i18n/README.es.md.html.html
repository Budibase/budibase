  <h3>
    <a href="#organizacion-del-repositorio" name="organizacion-del-repositorio" class="pilcrow"></a>
Organizacion del repositorio
  </h3>
</div>
<p>Budibase es un repositorio unico gestionado por Lerna. Lerna construye y publica los paquetes de Budibase sincronizandolos
cada ves que se realiza un cambio. A rasgos generales, estos son los paquetes que conforman Budibase:</p>
<ul>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/builder">packages/builder</a> - contiene el codigo del builder de la parte cliente, esta es una aplicacion svelte.</p>
</li>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/client">packages/client</a> - Este modulo se ejecuta en el browser y es el responsable de leer definiciones JSON y crear aplicaciones web en el momento.</p>
</li>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/server">packages/server</a> - La parte servidor de Budibase. Esta aplicacion Koa es responsable de suministrar lo necesario al builder para asi generar las aplicaciones Budibase. Tambien provee una API para interaccionar con la base de datos y el almacenamiento de ficheros.</p>
</li>
</ul>
<p>Para mas informacion, por favor lee el siguiente documento <a href="https://github.com/Budibase/budibase/blob/HEAD/docs/CONTRIBUTING.md">CONTRIBUTING.md</a></p>
