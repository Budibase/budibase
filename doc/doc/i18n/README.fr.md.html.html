  </h3>

<div class="pilwrap" id="construire-et-deployer-un-vrai-logiciel">
  <h3>
    <a href="#construire-et-deployer-un-vrai-logiciel" name="construire-et-deployer-un-vrai-logiciel" class="pilcrow"></a>
Construire et déployer un vrai logiciel
  </h3>
</div>
<p>Contrairement à d'autres plateformes, avec Budibase vous construisez et déployez des applications one-page. Les applications Budibase sont très perfomantes et peuvent être designées de manière responsive, offrant ainsi à vos utilisateurs une expérience exceptionnelle.
<br /><br /></p>
<div class="pilwrap" id="source-libre-et-extensible">
  <h3>
    <a href="#source-libre-et-extensible" name="source-libre-et-extensible" class="pilcrow"></a>
Source libre et extensible
  </h3>
</div>
<p>Budibase est un logiciel libre - sous licence GPL v3. Cela devrait vous rassurer sur le fait que Budibase sera toujours là. Vous pouvez également coder dans Budibase ou le forker et apporter des modifications à votre guise, ce qui en fera une expérience conviviale pour les développeurs.
<br /><br /></p>
<div class="pilwrap" id="importer-les-donnees-ou-partir-de-zero">
  <h3>
    <a href="#importer-les-donnees-ou-partir-de-zero" name="importer-les-donnees-ou-partir-de-zero" class="pilcrow"></a>
Importer les données ou partir de zéro
  </h3>
</div>
<p>Budibase peut tirer ses données de plusieurs sources, dont MongoDB, CouchDB, PostgreSQL, MySQL, Airtable, S3, DynamoDB ou une API REST. Et contrairement à d'autres plateformes, avec Budibase, vous pouvez partir de zéro et créer des applications métier sans aucune source de données. <a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">Demander une nouvelle source de données</a>.</p>
<p align="center">
  <img alt="Budibase data" src="https://res.cloudinary.com/daog6scxm/image/upload/v1636970242/Out%20of%20beta%20launch/data_n1tlhf.png">
</p>
<br /><br />
<div class="pilwrap" id="concevoir-et-creer-des-applications-a-laide-de-composants-predefinis">
  <h3>
    <a href="#concevoir-et-creer-des-applications-a-laide-de-composants-predefinis" name="concevoir-et-creer-des-applications-a-laide-de-composants-predefinis" class="pilcrow"></a>
Concevoir et créer des applications à l'aide de composants prédéfinis.
  </h3>
</div>
<p>Budibase est livré avec des composants joliment conçus et puissants que vous pouvez utiliser comme des blocs de construction pour bâtir votre interface utilisateur. Nous exposons également un grand nombre de vos options de style CSS préférées afin que vous puissiez faire preuve d'une créativité accrue. <a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">Demander un nouveau composant</a>.</p>
<p align="center">
  <img alt="Budibase design" src="https://res.cloudinary.com/daog6scxm/image/upload/v1636970243/Out%20of%20beta%20launch/design-like-a-pro_qhlfeu.gif">
</p>
<br /><br />
<div class="pilwrap" id="automatiser-les-processus-integrer-dautres-outils-et-se-connecter-a-des-webhooks">
  <h3>
    <a href="#automatiser-les-processus-integrer-dautres-outils-et-se-connecter-a-des-webhooks" name="automatiser-les-processus-integrer-dautres-outils-et-se-connecter-a-des-webhooks" class="pilcrow"></a>
Automatiser les processus, intégrer d'autres outils et se connecter à des webhooks
  </h3>
</div>
<p>Gagnez du temps en automatisant les processus manuels et les flux de travail. Qu'il s'agisse de se connecter à des webhooks ou d'automatiser des e-mails, il suffit de dire à Budibase ce qu'il doit faire et de le laisser travailler pour vous. Vous pouvez aisément <a href="https://github.com/Budibase/automations">créer une nouvelle automatisation pour Budibase ici</a> ou <a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">Demander une nouvelle automatisation</a>.</p>
<p align="center">

  <h3>
    <a href="#integration-avec-vos-outils-preferes" name="integration-avec-vos-outils-preferes" class="pilcrow"></a>
Intégration avec vos outils préférés
  </h3>
</div>
<p>Budibase s'intègre à un certain nombre d'outils populaires, ce qui vous permet de créer des applications qui s'adaptent parfaitement à votre pile technologique.</p>

<div class="pilwrap" id="paradis-des-admins">
  <h3>
    <a href="#paradis-des-admins" name="paradis-des-admins" class="pilcrow"></a>
Paradis des admins
  </h3>
</div>
<p>Budibase est conçu pour évoluer. Avec Budibase, vous pouvez vous auto-héberger sur votre propre infrastructure et gérer globalement les utilisateurs, l'accueil, le SMTP, les applications, les groupes, l'apparence et plus encore. Vous pouvez également fournir aux utilisateurs/groupes un portail d'applications et confier la gestion des utilisateurs au responsable du groupe.</p>
<ul>
<li>Regardez la vidéo de promotion: https://youtu.be/xoljVpty_Kw</li>
</ul>

<div class="pilwrap" id="comment-le-repo-est-il-organise">
  <h3>
    <a href="#comment-le-repo-est-il-organise" name="comment-le-repo-est-il-organise" class="pilcrow"></a>
Comment le repo est-il organisé ?
  </h3>
</div>
<p>Budibase est une monorepo gérée par lerna. Lerna gère la construction et la publication des paquets de Budibase. Voici, à un haut niveau, les paquets qui composent Budibase.</p>
<ul>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/builder">packages/builder</a> - contient le code pour l'application svelte côté client du budibase builder.</p>
</li>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/client">packages/client</a> - Un module qui s'exécute dans le navigateur et qui est chargé de lire les définitions JSON et de créer des applications web vivantes à partir de celles-ci..</p>
</li>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/server">packages/server</a> - Le serveur budibase. Cette application Koa est responsable de servir le JS pour les applications builder et budibase, ainsi que de fournir l'API pour l'interaction avec la base de données et le système de fichiers.</p>
</li>
</ul>
<p>Pour plus d'informations, voir <a href="https://github.com/Budibase/budibase/blob/HEAD/.github/CONTRIBUTING.md">CONTRIBUTING.md</a></p>
