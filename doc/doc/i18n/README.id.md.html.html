  </h3>
</div>
<p align="center">
  Budibase adalah sebuah platform <i>low-code</i> dengan sumber terbuka, bertujuan untuk membantu para <i>developer</i> dan professional IT, untuk membangun, mengotomatisasi, dan mengirimkan aplikasi internal di infrastruktur sendiri dalam waktu hitungan menit.
</p>

<div class="pilwrap" id="sparkles-fitur-fitur">
  <h2>
    <a href="#sparkles-fitur-fitur" name="sparkles-fitur-fitur" class="pilcrow"></a>
âœ¨ Fitur-fitur
  </h2>
</div>
<ul>
<li>
<p><strong>Membangun dan mengirimkan perangkat lunak sebenarnya .</strong> Tidak seperti platform lain, dengan Budibase Anda membuat dan mengirimkan aplikasi-aplikasi satu halaman. Aplikasi-aplikasi Budibase memiliki kinerja yang matang dan dapat dirancang secara responsif, memberikan pengalaman yang menyenangkan bagi pengguna.</p>
</li>
<li>
<p><strong>Sumber terbuka dan dapat dikembangkan.</strong> Budibase adalah platform dengan sumber terbuka - dilisensikan sebagai GPL v3. Ini akan membuat Anda yakin bahwa Budibase akan selalu ada. Anda juga dapat berkontribusi terhadap kode Budibase atau melakukan <i>fork</i> dan membuat perubahan sesuka Anda, memberikan pengalaman yang ramah developer.</p>
</li>
<li>
<p><strong>Memuat data atau mulai dari awal.</strong> Budibase menarik data dari berbagai sumber, termasuk MongoDB, CouchDB, PostgreSQL, mySQL, Airtable, S3, DyanmoDB, atau REST API. Dan tidak seperti platform lain, dengan Budibase Anda dapat memulai dari awal dan membuat aplikasi bisnis tanpa sumber data sekalipun. <a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">Permintaan untuk sumber data baru</a>.</p>
</li>
<li>
<p><strong>Rancang dan bangun aplikasi dengan komponen siap pakai yang kuat.</strong> Budibase tersedia dengan siap pakai dan dirancang dengan indah, komponen-komponen kuat yang dapat anda gunakan layaknya blok penyusun untuk merancang antar muka Anda. Kami juga menyediakan banyak sekali pilihan-pilihan CSS favorit yang dapat digunakan sesuai dengan kreatifitas yang Anda inginkan. [Permintaan komponen baru <a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">Permintaan untuk komponen baru</a>.</p>
</li>
<li>
<p><strong>Otomatiskan proses, integrasikan dengan aplikasi lain, dan sambungkan ke webhook.</strong> Menghemat waktu dengan mengotomatisasikan proses dan alur kerja yang masih dikerjakan secara manual. Dari menghubungkan ke webhooks, hingga mengotomatiasi surel, cukup beritahu Budibase apa yang harus dilakukan dan biarkan dia berkerja untuk Anda. Anda bisa dengan mudah <a href="https://github.com/Budibase/automations">membuat otomatisasi baru untuk Budibase disini </a> atau <a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">minta otomatisasi baru</a>.</p>
</li>
<li>
<p><strong>Surga bagi Admin.</strong> Budibase dibuat untuk berkembang. Dengan Budibase, Anda dapat meng<i>hosting</i> sendiri di infrastruktrur Anda punya dan secara global mengelola para pengguna, orientasi, SMTP, aplikasi-aplikasi, grup-grup, tema, dan masih masih banyak lagi. Anda juga dapat menyediakan para pengguna/grup dengan sebuah aplikasi portal dan menyebarkan pengelolaan pengguna kepada manaajer grup.</p>
</li>
</ul>

<div class="pilwrap" id="memulai">
  <h2>
    <a href="#memulai" name="memulai" class="pilcrow"></a>
ğŸ Memulai
  </h2>
</div>
<p>Saat ini ada 2 cara untuk memulai Budibase, yaitu: Digital Ocean dan Docker.
<br /><br /></p>
<div class="pilwrap" id="memulai-dengan-digital-ocean">
  <h3>
    <a href="#memulai-dengan-digital-ocean" name="memulai-dengan-digital-ocean" class="pilcrow"></a>
Memulai Dengan Digital Ocean
  </h3>
</div>
<p>Cara paling mudah dan cepat untuk memulai adalah dengan menggunakan Digital Ocean:
<a href="https://marketplace.digitalocean.com/apps/budibase">1-klik Digital Ocean <i>deploy</i></a></p>
<a href="https://marketplace.digitalocean.com/apps/budibase">
  <img src="https://user-images.githubusercontent.com/552074/87779219-5c3b7600-c824-11ea-9898-981a8ba94f6c.png" alt="digital ocean badge">
</a>  
<br /><br />
<div class="pilwrap" id="memulai-dengan-docker">
  <h3>
    <a href="#memulai-dengan-docker" name="memulai-dengan-docker" class="pilcrow"></a>
Memulai dengan Docker
  </h3>
</div>
<p>Untuk memulai, Anda harus memiliki docker dan docker compose ter<i>install</i> di mesin anda.
Setelah Anda docker ter<i>install</i>, prosesnya akan memakan waktu 5 menit, dengan empat langkah ini:</p>
<ol>
<li><i>Install</i> CLI Budibase.</li>
</ol>
<pre><code>$ npm i -g @budibase/cli 
</code></pre>
<ol start="2">
<li>Mempersiapkan Budibase (pilih tempat menyimpan Budibase dan <i>port</i> untuk menjalankannya)</li>
</ol>
<pre><code>budi hosting --init 
</code></pre>
<ol start="3">
<li>Jalankan Budibase</li>
</ol>
<pre><code>budi hosting --start 
</code></pre>
<ol start="4">
<li>Buat pengguna admin Anda</li>
</ol>
<p>Masukkan surel dan sandi untuk admin baru.</p>
<p>Selesai! Anda sekarang siap untuk membangun aplikasi internal yang mumpuni dalam hitungan menit. Untuk informasi tambahan tentang cara memulai dan mempelajari Budibase, kunjungi <a href="https://docs.budibase.com/getting-started">dokumentasi</a> kami.</p>

<div class="pilwrap" id="bagaimana-repositori-diatur">
  <h3>
    <a href="#bagaimana-repositori-diatur" name="bagaimana-repositori-diatur" class="pilcrow"></a>
Bagaimana repositori diatur
  </h3>
</div>
<p>Budibase adalah <i>monorepo</i> yang dikelola oleh lerna. Lerna mengelola pembangunan dan penerbitan paket Budibase. Pada level tinggi, inilah paket-paket yang membentuk Budibase.</p>
<ul>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/builder">packages/builder</a> - berisi kode untuk aplikasi svelte sisi klien pembangun Budibase.</p>
</li>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/client">packages/client</a> - Modul yang berjalan di <i>browser</i> yang bertanggung jawab untuk membaca definisi JSON dan membuat aplikasi web yang hidup dan bernafas darinya.</p>
</li>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/server">packages/server</a> - <i>Server</i> Budibase. Aplikasi Koa ini bertanggung jawab untuk melayani JS untuk aplikasi builder dan budibase, serta menyediakan API untuk interaksi dengan database dan sistem file.</p>
</li>
</ul>
<p>Untuk informasi lebih lanjut, lihat <a href="https://github.com/Budibase/budibase/blob/HEAD/.github/CONTRIBUTING.md">CONTRIBUTING.md</a>
