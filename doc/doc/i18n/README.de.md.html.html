<div class="pilwrap" id="entwickle-automatisiere-und-stelle-interne-tools-in-minuten-bereit.">

  Entwickle, automatisiere und stelle interne Tools in Minuten bereit.

</div>
<p align="center">

</p>

<div class="pilwrap" id="sparkles-features">
  <h2>
    <a href="#sparkles-features" name="sparkles-features" class="pilcrow"></a>
✨ Features
  </h2>
</div>
<ul>
<li>
<p><strong>Entwickle echte Webanwendungen.</strong> Anders als ähnliche Plattformen entwickelst du mit Budibase echte Single-Page Webapplikationen (SPAs). Deine Budibase-Apps sind standardmäßig hochperformant und haben ein Responsive-Design für eine großartige Benutzererfahrung.</p>
</li>
<li>
<p><strong>Quelloffen und erweiterbar.</strong> Budibase ist quelloffen - lizenziert unter der GPL v3. Du kannst darauf vertrauen, dass Budibase auch in der Zukunft immer zur Verfügung steht. Budibase bietet eine Entwicklerfreundliche Plattform: du kannst Budibase erweitern, oder die Codebase forken und eigene Änderungen vornehmen.</p>
</li>
<li>
<p><strong>Datenquellen einbinden oder von Null starten.</strong> Budibase kann Daten aus vielen Quellen einbinden, unter anderem aus MongoDB, CouchDB, PostgreSQL, MySQL, Airtable, S3, DynamoDB, oder einer REST API. Und anders als ähnliche Plattformen erlaubt Budibase auch die App-Entwicklung komplett ohne Datenquellen mit einer internen Datenbank. Deine Datenquelle noch nicht dabei? <a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">Frag einfach nach</a>.</p>
</li>
<li>
<p><strong>Designe und entwickle Apps mit leistungsfähigen Komponenten.</strong> Budibase kommt fertig mit optisch ansprechenden und leistungsfähigen Komponenten, die als Bausteine für deine UI dienen. Außerdem kannst du die UI mit vielen CSS-Styles nach deinem Geschmack anpassen. Fehlt dir eine Komponente? <a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">Frag uns hier</a>.</p>
</li>
<li>
<p><strong>Automatisiere Prozesse, integriere andere Tools und binde Web-APIs ein.</strong> Spar dir Zeit, indem du manuelle Prozesse einfach automatisierst: Vom Verbinden mit Web-Hooks bis zum automatischen Senden von E-Mails, Budibase kann alles für dich erledigen. Eine Automatisierung ist noch nicht dabei? Du kannst einfach <a href="https://github.com/Budibase/automations">deine eigene erstellen</a> oder <a href="https://github.com/Budibase/budibase/discussions?discussions_q=category%3AIdeas">uns deine Idee mitteilen</a>.</p>
</li>
<li>
<p><strong>Ein Paradies für Systemadministratoren</strong> Budibase ist von Grund auf für das Skalieren ausgelegt. Du kannst Budibase einfach auf deiner eigenen Infrastruktur hosten und global Benutzer, Onboarding, SMTP, Applikationen, Gruppen, UI-Themes und mehr verwalten. Du kannst außerdem ein übersichtliches App-Portal für deine Benutzer bereitstellen und das Benutzermanagement an Gruppen-Manager delegieren.</p>
</li>
</ul>

<div class="pilwrap" id="los-gehts">
  <h2>
    <a href="#los-gehts" name="los-gehts" class="pilcrow"></a>
🏁 Los geht's
  </h2>
</div>
<p>Momentan existieren zwei Optionen mit Budibase loszulegen: Digital Ocean und Docker.
<br /><br /></p>
<div class="pilwrap" id="los-gehts-mit-digital-ocean">
  <h3>
    <a href="#los-gehts-mit-digital-ocean" name="los-gehts-mit-digital-ocean" class="pilcrow"></a>
Los geht's mit Digital Ocean
  </h3>
</div>
<p>Der einfachste und schnellste Weg loszulegen ist Digital Ocean:
<a href="https://marketplace.digitalocean.com/apps/budibase">1-Click Deploy auf Digital Ocean</a></p>
<a href="https://marketplace.digitalocean.com/apps/budibase">
  <img src="https://user-images.githubusercontent.com/552074/87779219-5c3b7600-c824-11ea-9898-981a8ba94f6c.png" alt="digital ocean badge">
</a>  
<br /><br />
<div class="pilwrap" id="los-gehts-mit-docker">
  <h3>
    <a href="#los-gehts-mit-docker" name="los-gehts-mit-docker" class="pilcrow"></a>
Los geht's mit Docker
  </h3>
</div>
<p>Um loszulegen musst du bereits <code>docker</code> und <code>docker compose</code> auf deinem Computer installiert haben.
Sobald du Docker installiert hast brauchst du ca. 5 Minuten für diese 4 Schritte:</p>
<ol>
<li>Installiere das Budibase CLI Tool.</li>
</ol>
<pre><code>$ npm i -g @budibase/cli 
</code></pre>
<ol start="2">
<li>Installiere Budibase (wähle den Speicherort und den Port auf dem Budibase laufen soll.)</li>
</ol>
<pre><code>$ budi hosting --init 
</code></pre>
<ol start="3">
<li>Führe Budibase aus.</li>
</ol>
<pre><code>$ budi hosting --start 
</code></pre>
<ol start="4">
<li>Lege einen Admin-Benutzer an.
Gib die E-Mail und das Passwort für den neuen Admin-Benutzer ein.</li>
</ol>
<p>Schon geschafft! Jetzt kann es losgehen mit der minutenschnellen Entwicklung deiner Tools. Für weitere Informationen und Tipps schau doch mal in unsere <a href="https://docs.budibase.com/docs/quickstart-tutorials">Dokumentation</a>.</p>

<div class="pilwrap" id="wie-die-repository-strukturiert-ist">
  <h3>
    <a href="#wie-die-repository-strukturiert-ist" name="wie-die-repository-strukturiert-ist" class="pilcrow"></a>
Wie die Repository strukturiert ist.
  </h3>
</div>
<p>Budibase ist eine Monorepo, die von Lerna verwaltet wird. Lerna verwaltet das Erstellen und Veröffentlichen von Budibase-Paketen.
Grob besteht Budibase aus folgenden Modulen:</p>
<ul>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/builder">packages/builder</a> - enthält Code für den clientseitigen Budibase Builder, mit dem Anwendungen erstellt werden.</p>
</li>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/client">packages/client</a> - Ein Modul, das im Browser läuft und aus JSON-Definitionen funktionsfähige Web-Apps erstellt.</p>
</li>
<li>
<p><a href="https://github.com/Budibase/budibase/tree/HEAD/packages/server">packages/server</a> - Der Budibase Server. Diese Koa-Anwendung stellt den Javascript-Code für den Builder und den Client bereit, und bietet eine API für die Interaktion mit dem Budibase Backend, Datenbanken und dem Dateisystem.</p>
</li>
</ul>
<p>Für mehr Informationen schau in die <a href="https://github.com/Budibase/budibase/blob/HEAD/.github/CONTRIBUTING.md">CONTRIBUTING.md</a>
<br /><br /></p>
