Create `scripts/generate-hardened-dockerfile.sh` — a bash script that generates all 3 Dockerfile.hardened files from their source Dockerfiles using sed transformations. Follow the pattern established by `scripts/generate-hardened-package.sh` (per D-02).

The script structure:

```bash
#!/bin/bash
set -euo pipefail
REPO_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
```

**generate_alpine() function** — handles server and worker (per D-01, D-03, D-06):

Takes two args: source Dockerfile path, package directory name (e.g. "packages/server").

Three sed transformations applied in order:

1. **OS patch insertion (OS-01, OS-02, OS-03):** Insert BEFORE the `# handle node-gyp` comment line (NOT after FROM — see Pitfall 5 in RESEARCH.md). The inserted block is:
```
# OS-level CVE patches — scoped to named packages only (OS-01, OS-02, OS-03)
# zlib 1.3.2-r0: fixes CVE-2026-22184 (CRITICAL) — Alpine 3.23 stable
# curl 8.19.0-r0: fixes CVE-2026-3805 (HIGH) — requires Alpine edge/main
RUN apk add --upgrade --no-cache \
    --repository=https://dl-cdn.alpinelinux.org/alpine/edge/main \
    zlib \
    curl

```
Use sed `/# handle node-gyp/i\` to insert before that comment line. The server Dockerfile has `# handle node-gyp and install postgres client for pg_dump utils` at line 18. The worker has `# handle node-gyp` at line 9. Use the pattern `# handle node-gyp` which matches both.

2. **package.hardened.json substitution (DOCK-06 preservation):** Replace the COPY package.json line with COPY package.hardened.json. For server: `s|COPY packages/server/package.json \.|COPY packages/server/package.hardened.json ./package.json|`. For worker: `s|COPY packages/worker/package.json \.|COPY packages/worker/package.hardened.json ./package.json|`. The `./package.json` target ensures the file is named correctly in the container.

3. **Non-root USER insertion (DOCK-05, per D-04):** Insert BEFORE the CMD line. The inserted block is:
```
# Drop to non-root user (DOCK-05)
# node user (UID 1000) pre-exists in node:22-alpine
RUN chown -R node:node /app
USER node

```
Use sed `/^CMD /i\` to insert before the CMD line. This placement is AFTER all COPY instructions (see Pitfall 6 in RESEARCH.md).

4. **Native addon smoke test:** Insert AFTER the last COPY line and BEFORE the chown/USER block. Add:
```
# Smoke test: verify native addons survive OS package upgrade
RUN node -e "require('bcrypt'); require('isolated-vm'); console.log('native addons OK')"
```
This catches ABI breakage from edge repo OpenSSL upgrades at build time (Pitfall 2).

5. **Add a header comment** as the first line: `# AUTO-GENERATED by scripts/generate-hardened-dockerfile.sh — do not edit manually`

**generate_couchdb() function** — handles CouchDB (per D-05, D-07):

Takes one arg: source Dockerfile path.

The CouchDB Dockerfile already has `apt-mark hold couchdb && apt-get update && apt-get upgrade -y` at lines 123-126 of the runner stage. The hardened variant needs minimal changes:
- Copy the entire file as-is (per research: "copy + add a generation comment header only")
- Add the same header comment as Alpine files
- Do NOT add a USER directive (per D-05 — CouchDB runner.sh needs root for chown)

**Main script body:**

```bash
generate_alpine "$REPO_ROOT/packages/server/Dockerfile" "packages/server"
generate_alpine "$REPO_ROOT/packages/worker/Dockerfile" "packages/worker"
generate_couchdb "$REPO_ROOT/hosting/couchdb/Dockerfile"
```

**Post-generation assertions (built into the script):**

After generating each file, assert:
- `grep -q 'package.hardened.json' "$OUT"` for server and worker (Pitfall 3)
- `grep -q 'removeWorkspaceDependencies' "$OUT"` for server and worker (DOCK-06)
- `grep -q 'USER node' "$OUT"` for server and worker (DOCK-05)
- `grep -q 'apk add --upgrade' "$OUT"` for server and worker (OS-01, OS-02)
- `grep -q 'alpine/edge/main' "$OUT"` for server and worker (OS-02)
- `grep -q 'apt-mark hold couchdb' "$OUT"` for couchdb (D-07)

If any assertion fails, exit 1 with a descriptive error.

Make the script executable: `chmod +x scripts/generate-hardened-dockerfile.sh`
