**Build all 3 images (per DOCK-01, DOCK-02, DOCK-03):**

```bash
# Server (DOCK-01, DOCK-04)
docker build -f packages/server/Dockerfile.hardened \
  --build-arg BUDIBASE_VERSION=test-hardened \
  --build-arg GIT_COMMIT_SHA=test \
  -t budibase/apps:test-hardened \
  .

# Worker (DOCK-02, DOCK-04)
docker build -f packages/worker/Dockerfile.hardened \
  --build-arg BUDIBASE_VERSION=test-hardened \
  --build-arg GIT_COMMIT_SHA=test \
  -t budibase/worker:test-hardened \
  .

# CouchDB (DOCK-03)
docker build -f hosting/couchdb/Dockerfile.hardened \
  -t budibase/couchdb:test-hardened \
  hosting/couchdb/
```

Each build MUST exit 0. The native addon smoke test (`require('bcrypt'); require('isolated-vm')`) runs inline during the server build — if it fails, the build itself fails.

**Functional verification after builds succeed:**

1. **OS package versions (OS-01, OS-02):**
```bash
# Server
docker run --rm budibase/apps:test-hardened sh -c 'apk list --installed 2>/dev/null | grep zlib'
# Must show zlib-1.3.2-r0 or higher
docker run --rm budibase/apps:test-hardened curl --version | head -1
# Must show curl 8.19.0 or higher

# Worker (same checks)
docker run --rm budibase/worker:test-hardened sh -c 'apk list --installed 2>/dev/null | grep zlib'
docker run --rm budibase/worker:test-hardened curl --version | head -1
```

2. **Non-root USER (DOCK-05):**
```bash
docker run --rm budibase/apps:test-hardened id
# Must show uid=1000(node)
docker run --rm budibase/worker:test-hardened id
# Must show uid=1000(node)
# CouchDB: do NOT check — must remain root (per D-05)
```

3. **Native addon integrity:**
```bash
docker run --rm budibase/apps:test-hardened node -e "require('bcrypt'); require('isolated-vm'); console.log('OK')"
# Must print OK
```

4. **npm CVE resolution versions (Phase 1 integration):**
```bash
docker run --rm budibase/apps:test-hardened node -e "console.log(require('fast-xml-parser/package.json').version)"
# Must be >= 4.5.4
docker run --rm budibase/apps:test-hardened node -e "console.log(require('axios/package.json').version)"
# Must be >= 1.13.5
```

If any check fails, diagnose by reading the Dockerfile.hardened and build output, fix the generation script if needed, regenerate, and rebuild.
